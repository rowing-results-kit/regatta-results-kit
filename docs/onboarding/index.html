regatta-results-kit — 協会担当者向けセットアップ手順

テンプレートをコピーして始める

GitHubとGoogleがあれば、
自力で速報サイトを立ち上げられます。

このページを見ながら7つのステップを進めるだけで、大会当日に使える速報サイトが完成します。 ターミナル操作・プログラミング知識は一切不要です。

Step 1 から始める

全体の流れ — 7ステップ

  • 1 テンプレートをコピー 3〜5分
  • 2 Cloudflare Pages に接続 10〜15分
  • 3 大会情報を入力して初期構築 2〜5分
  • 4 Google Drive フォルダを作成 5分
  • 5 GAS テンプレートをコピー・設定 10〜20分
  • 6 GitHub PAT を作成 10分
  • 7 管理ポータルで接続テスト 10分

合計：約50〜70分

1

テンプレートリポジトリをコピーする

3〜5分

操作手順

  1. 1 github.com/rowing-results-kit/regatta-results-kit を開く
  2. 2 右上の 「Use this template」（緑ボタン）→ 「Create a new repository」をクリック
  3. 3 Owner（所有者）を自団体のアカウントに変更する
  4. 4 Repository name（リポジトリ名）を入力する（例：association-regatta-2027）
  5. 5 公開範囲は 「Private」（非公開）を選ぶ
  6. 6 「Create repository」をクリック
先に GitHub にログインしてから「Use this template」を押してください。未ログインのままボタンを押すとログイン画面に飛ばされます。
リポジトリとは：ファイルをまとめて管理するGitHubのフォルダのことです。テンプレートをコピーすることで、速報サイトの雛形が自分のアカウントに作られます。

Cloudflare Pages に接続する

10〜15分

操作手順

  1. 1 dash.cloudflare.com を開いてログイン（アカウントがなければ無料で新規作成）
  2. 2 左メニュー → Workers & Pages → Pages → 「Create application」 → 「Connect to Git」
  3. 3 「Connect GitHub」でGitHubを連携 → Step 1 で作成したリポジトリを選択 → 「Begin setup」
  4. 4 ビルド設定を右の表のとおりに入力する（特に Build output directory が重要）
  5. 5 「Save and Deploy」をクリックして2〜5分待つ
Cloudflareとは：無料で使えるウェブサイト公開サービスです。GitHubのファイルを自動的にウェブサイトとして公開してくれます。

Q 詰まったとき

サイトが 404 または真っ白になる

原因：Build output directory が違います。Cloudflare Pages の Settings → Builds & deployments → Build output directory を site に修正して「Save and Deploy」してください。

Deployments が「Failed」になる

原因：Build command に何か入力されています。Build command を空欄にして再デプロイしてください。

GitHub Actions で大会情報を入力して初期構築する

2〜5分
ターミナル・Pythonは不要です。GitHubの画面だけで操作できます。

操作手順

  1. 1 Step 1 で作成したリポジトリを GitHub で開く
  2. 2 上部タブの 「Actions」 をクリック
  3. 3 左サイドバーの 「大会初期構築」（setup-tournament）をクリック
  4. 4 右側の 「Run workflow」 ボタンをクリック
  5. 5 フォームに大会名・会場・日付・カラーを入力する（右の入力例を参考に）
  6. 6 青い 「Run workflow」 ボタンをクリックして実行

詰まった場合: Claude に相談する コピーしました

Q 詰まったとき

「大会初期構築」が Actions の左側に出てこない

対処：テンプレートのコピーが古い可能性があります。Step 1 からやり直してください。

ジョブが赤バツ（✗）になる

エラーの詳細を見る手順：
Actions タブの赤バツのジョブ行をクリック → 左の「scaffold 実行」等の行をクリック → ログが展開されて日本語のエラー理由が表示されます。
よくある原因：日付の形式が違う（スラッシュ区切り推奨、例: 2026/06/14。ハイフン区切り 2026-06-14 も可）、またはカラーコードが違う（「#」から始まる7文字、例: #1a3a5c）。修正してから「Run workflow」を再実行してください。

github.com/<org>/<repo>/actions/runs/...
✗ 大会初期構築 — 失敗
✗ scaffold 実行 ← ここをクリックでログ表示
Error: 開始日の形式が正しくありません。例: 2026/06/14（ハイフン 2026-06-14 も可）

site/data/master.json が作られない

対処：リポジトリの Settings → Actions → General → Workflow permissions を 「Read and write permissions」 に変更してから再実行してください。

Google Drive にフォルダを作成する

5分

操作手順

  1. 1 Google Drive（自団体のGoogleアカウント）を開く
  2. 2 「新規」→「フォルダ」でルートフォルダを1つ作成（名前は任意。例：○○レガッタ2027）
  3. 3 作成したフォルダをダブルクリックして開く
  4. 4 ブラウザのアドレスバー（URL）を確認し、末尾の英数字（フォルダID）をコピーする
フォルダIDとは：Google Drive のURLの末尾にある英数字の文字列です。次のステップで使います。
https://drive.google.com/drive/folders/1AbCdEfGhIjKlMnOpQrStU
← この部分（太字の英数字）をコピー
サブフォルダは GAS の初回実行時に自動で作成されます。今は作らなくて大丈夫です。

GAS テンプレートをコピーしてプロパティを設定する

10〜20分
GAS（Google Apps Script）とは：Googleが提供する無料の自動化ツールです。今回はCSVデータを速報サイトに自動送信する役割を担います。コードの貼り付けは不要です。

GAS-A（必須）の操作手順

  1. 1 下の「テンプレートを開く」ボタンをクリック（GAS テンプレートが開きます）
  2. 2 メニュー 「ファイル」→「コピーを作成」 → 名前を入力 → 「コピーを作成」
  3. 3 コピーされた GAS プロジェクトが開いたら、左側の 歯車アイコン（プロジェクトの設定） をクリック
  4. 4 「スクリプトプロパティ」 セクションで以下の2つのプロパティを入力する
プロパティ名（キー） 設定する値
DRIVE_ROOT_FOLDER_ID必須 Step 4 でコピーしたフォルダID
GITHUB_TOKEN Step 6 が完了してから入力（後でOK）
GitHub リポジトリ設定・計測ポイントについて:
GITHUB_OWNER / GITHUB_REPO / MEASUREMENT_POINTS は Step 7 の管理者ポータル「接続設定」タブで設定するため、ここでのスクリプトプロパティ手入力は不要です。GitHub リポジトリの URL を貼り付けるだけで自動設定され、計測ポイントは 500m,1000m で自動固定されます。

GitHub PAT（アクセストークン）を作成して設定する

10分
重要：PAT（Personal Access Token）は一度しか表示されません。必ずコピーしてから画面を閉じてください。
PAT（Personal Access Token）とは：GAS から GitHub に書き込みをするための「鍵」です。パスワードの代わりに使う文字列で、90日間有効です。

操作手順

  1. 1 GitHub にログイン → 右上アバター → 「Settings」
  2. 2 左メニュー最下部の 「Developer settings」 → 「Personal access tokens」 → 「Fine-grained tokens」 → 「Generate new token」
  3. 3 設定を右の表のとおりに入力する
  4. 4 「Generate token」をクリック → 表示された github_pat_... で始まる文字列を今すぐコピー
  5. 5 GAS-A のスクリプトプロパティ → GITHUB_TOKEN にペーストして保存
項目 設定値
Token name 任意（例: regatta-2027-gas）
Expiration 90 days
Repository access Only select repositories → Step 1 のリポジトリのみ選択
Repository permissions → Contents Read and Write必須
それ以外の権限 No access のまま（触らない）

7

管理者ポータルを公開して接続テストをする

10分

操作手順

  1. 1 GAS-A エディタを開く → 右上 「デプロイ」 → 「新しいデプロイ」
  2. 2 歯車アイコン → 「ウェブアプリ」 を選択
  3. 3 「次のユーザーとして実行」→ 自分、「アクセスできるユーザー」→ 自分のみ に設定
  4. 4 「デプロイ」 → 権限確認 → 「許可」
  5. 5 表示された ウェブアプリURL をブックマークして開く → 管理者ポータルが表示される
  6. 6 「初期セットアップ実行」ボタンを押す（書類フォルダの自動作成と2分ごとの自動更新が開始されます。何度押しても安全です）
  7. 7 「GitHub 接続テスト」 をクリック → 緑バッジが出れば OK
  8. 8 「Drive 接続テスト」 をクリック → フォルダ名が表示されれば OK
  9. 9 状態が全て緑（自動更新: 稼働中 を含む）になったら導入完了です

Q 詰まったとき

「初期セットアップ実行」ボタンが見当たらない

対処：GAS-A のバージョンが古い可能性があります。Step 5 からやり直してください。

GitHub 接続テストが赤バッジになる

対処：GITHUB_TOKEN が間違っているか、または Step 7 管理者ポータル「接続設定」タブで GitHub リポジトリ URL が正しく設定されていません。GITHUB_TOKEN の値と管理者ポータルの設定を確認してから再テストしてください。

Drive テストが「DRIVE_ROOT_FOLDER_ID が未設定」と表示される

対処：Drive フォルダIDを確認して再入力 → 再テストしてください。

「You need permission」と表示される

原因：アクセス設定を「自分のみ」にしているため、別のGoogleアカウントではアクセスできません。正しいアカウントでアクセスしてください。
