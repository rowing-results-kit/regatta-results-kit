## CP-01 文字の重なり・見切れ・クリップ ゼロ

**なぜ**: 龍偉が繰り返し「文字が欠ける」「重なっている」と指摘。headless Chrome は最小幅 ~500px のため、モバイル幅では偽の見切れが発生する（2026-06-12 学習）。

**合格基準**:
- 320 / 375 / 414 / 768 / 1024 / 1280 / 1920px の全幅で bounding box 検査を通過
- `overflow: hidden` によるテキストクリップがない（または意図的で許容範囲内）
- 隣接要素との重なり（z-index 層含む）がない

**確認方法**: `node tools/checkpoint-qa.mjs --check overlap <対象>` で各幅のスクリーンショット+bounding box ログを出力。目視だけで終わらせない。

- [ ] 320px
- [ ] 375px
- [ ] 414px
- [ ] 768px
- [ ] 1024px
- [ ] 1280px
- [ ] 1920px

## CP-02 レスポンシブ（横スクロール・孤立文字ゼロ）

**なぜ**: スマホ閲覧者が多い大会サイトで横スクロールが発生したと指摘。

**合格基準**:
- 全幅で `document.body.scrollWidth <= window.innerWidth`（横スクロールバーなし）
- 1文字孤立（「を、」「る。」行末1文字折り返し）なし
- `max-width` は `min(値, 100%)` または `min-width: 0` で親超過なし

**確認方法**: `node tools/checkpoint-qa.mjs --check responsive <対象>` で横スクロール幅を数値出力。

- [ ] 横スクロール: 全幅で 0px
- [ ] 1文字孤立: なし（手動目視）

## CP-03 見出しの改行（意味区切り・孤立行ゼロ）

**なぜ**: 見出しが途中で折り返されて意味が崩れる・1単語だけ孤立する問題を指摘。

**合格基準**:
- 見出し（h1〜h4）は `white-space: nowrap` または `<wbr>` / `word-break` で意味区切りを制御
- どの幅でも最終行に1単語だけ残らない
- 内部折り返しで孤立行（1〜2文字のみの行）を作らない

**確認方法**: 各ブレークポイントのスクリーンショットで手動確認。特に 375px・768px が要注意。

- [ ] 375px で見出し孤立なし
- [ ] 768px で見出し孤立なし
- [ ] 1024px で見出し孤立なし

## CP-04 成果物スクショ（全ページ・切り取りなし）

**なぜ**: スクショが一部しか写っていない・object-fit:cover で端が切れていると指摘。公式様式は変更禁止（大会ルール）。

**合格基準**:

- `object-fit: contain`（cover 禁止。端を切ってはならない）
- HTML 再現でなく実物・実出力を使う
- 公式様式・フォーマットは一切変更しない

**確認方法**: `node tools/checkpoint-qa.mjs --screenshot <対象>` で full-page スクリーンショットを生成し目視確認。

- [ ] 全ページ full-page スクリーンショット取得済み
- [ ] object-fit: contain または画像クリップなし
- [ ] 公式フォーマット不変

## CP-05 絵文字ゼロ → モノクロ線アイコン統一

**なぜ**: 龍偉が「AIっぽい」と指摘する最大の原因が絵文字。favicon・ボタン・ラベルを含めて全排除。

**合格基準**:
- 絵文字（Unicode 範囲 U+1F000〜U+1FFFF 等）が HTML/CSS/JS に 0 件
- アイコンはモノクロ線アイコン（stroke ベース・単色）を使用
- favicon も SVG line icon か letter-based（絵文字系 PNG 禁止）

**確認方法**: `node tools/checkpoint-qa.mjs --check emoji <対象>` でソース全体を grep + DOM テキスト検査。

- [ ] HTML ソース内絵文字: 0件
- [ ] favicon: 絵文字系でない
- [ ] アイコン: stroke ベース or CSS/SVG

## CP-06 AIらしくないデザイン（DESIGN.md 禁止7項目）

**なぜ**: 龍偉が繰り返し「AIが作った感が出ている」と指摘。自然なデザインを要求。

**合格基準（禁止7項目 — 全て 0 件）**:
1. 絵文字（CP-05 と重複・最重要）
2. 全面丸ボタン（border-radius: 50% / 9999px の大型ボタン）
3. 影の多用（box-shadow が3箇所超）
4. 等幅3カラム羅列（grid-template-columns: 1fr 1fr 1fr のカード羅列）
5. 中央揃え連発（text-align: center が本文テキストに多用）
6. 「〜できます」連発（コピーライティング）
7. font-weight: 400 未満（細字）

**確認方法**: ソース内 CSS 検査 + 目視。

- [ ] 全面丸ボタンなし
- [ ] box-shadow 3箇所以内
- [ ] 3カラム等幅羅列なし（または意図的）
- [ ] 本文 text-align:center 多用なし
- [ ] font-weight ≥ 400

## CP-07 左端整列（ロゴ基準）

**なぜ**: 各要素の左端がバラバラで「揃っていない」と指摘。

**合格基準**:

- `padding-left` / `margin-left` が要素ごとに恣意的に異なっていない

**確認方法**: DevTools の element highlight で左端を目視比較。

- [ ] ロゴ・見出し・本文の左端が揃っている

## CP-11 アクセシビリティ（WCAG 2.1 AA 基準）

**なぜ**: Gate 1 の D-M2 でモバイルのアクセシビリティを受入条件に追加。

**合格基準**:
- テキストコントラスト比 ≥ 4.5:1（小テキスト）/ ≥ 3:1（大テキスト）
- `focus-visible` でキーボードフォーカスが視覚的に確認できる
- `aria-live` で動的コンテンツ更新をスクリーンリーダーに通知
- `<table>` の `<th>` に `scope` 属性あり
- タップターゲット ≥ 44×44px（iOS HIG 準拠）

**確認方法**: `node tools/checkpoint-qa.mjs --check a11y <対象>` でコントラスト比・aria 属性を検査。

- [ ] コントラスト比: 4.5:1 以上
- [ ] focus-visible: あり
- [ ] aria-live: 動的更新箇所に付与
- [ ] th[scope]: 全テーブルに付与
- [ ] タップターゲット: ≥ 44px
