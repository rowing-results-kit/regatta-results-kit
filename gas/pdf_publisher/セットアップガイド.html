マスターズPDF生成 セットアップガイド

試合結果PDFを2分ごとに自動生成するための初期設定です。上から順番に1つずつ進めてください。

初期セットアップ

手順 1

https://script.google.com を開きます。

手順 2

新しい Apps Script プロジェクトを作成します。

手順 3

プロジェクト名を マスターズPDF生成 に変更します。

手順 4

appsscript.json の内容を、このフォルダの appsscript.json と同じ内容に置き換えます。

手順 5

Code.gs の内容を、このフォルダの Code.gs と同じ内容に置き換えます。

手順 6

新しいファイル Setup.gs を作成し、このフォルダの Setup.gs の内容を貼り付けます。

手順 6-2

新しいファイル WebApp.gs を作成し、このフォルダの WebApp.gs の内容を貼り付けます。ブラウザのボタンから帳票を作るための画面です。

手順 6-3

「ファイル」の追加から HTML を選び、名前を webapp にして作成します。中身をすべて消してから、このフォルダの webapp.html の内容を貼り付けます。

HTML ファイルの名前は必ず webapp にしてください（webapp.html ではありません）。名前が違うと画面が開きません。

手順 7

プロジェクト設定を開き、スクリプトプロパティに次の5つを設定します。

  • GITHUB_REPO = {{GITHUB_REPO}}（例: owner/repo-name）
  • GITHUB_BRANCH = main
  • GITHUB_TOKEN = 既存GAS（CSV→JSON Push）で使用中のものと同じ値
  • TEMPLATE_SHEET_ID = {{TEMPLATE_SHEET_ID}}（Sheetsの雛形ID）
  • PDF_OUTPUT_FOLDER_ID = {{PDF_OUTPUT_FOLDER_ID}}（Drive フォルダID）

GITHUB_TOKEN が未設定だと GitHub API のレート制限でPDF生成が停止します。必ず既存GASと同じ Personal Access Token を設定してください。

手順 8

関数の一覧から saveSetup を選び、実行します。

手順 9

関数の一覧から initializeTemplate を選び、1回実行して雛形を初期化します。

これで雛形の順位列に 1〜6 がベタ書きされ、以降のレース処理が安定します。

手順 10

権限確認が出たら、使用するGoogleアカウントで承認します。

手順 11

関数の一覧から setupTrigger を選び、実行します。

手順 12

関数の一覧から testGenerateRace1 を選び、実行します。

  • testGenerateRace1RealFormat: 実 race_NNN.json 形式のダミーで動作確認
  • testGenerateRace1Status: DNS/DNF を含むダミーで備考欄表示確認
手順 13

Drive の結果フォルダに 1.pdf が作成されていることを確認します。

帳票をブラウザから作る（ウェブアプリ）

ここまでの設定で、結果ブックレット・結果一覧PDF・レース前準備資料を ブラウザのボタン から作れるようになります。スクリプトエディタで関数を選んで実行する必要はありません。

手順 14

画面右上の「デプロイ」→「新しいデプロイ」を選びます。

手順 15

「種類の選択」の歯車から ウェブアプリ を選びます。

手順 16

次のとおり設定して「デプロイ」を押します。

  • 次のユーザーとして実行: 自分
  • アクセスできるユーザー: 自分のみ

「アクセスできるユーザー」を 全員 にしないでください。帳票の作成が誰にでも実行できてしまいます。なお安全のため、誤って 全員 にした場合はページ側でも操作を拒否します。

手順 17

初回のみ、アクセスの承認を求められます。自分の Google アカウントを選び、内容を確認して許可します。

手順 18

表示された ウェブアプリの URL を開きます。ボタンを押すと帳票が作られ、Drive の出力フォルダに PDF が入ります。この URL はブックマークしておくと便利です。

コード（Code.gs / WebApp.gs / webapp）を更新したときは、「デプロイ」→「デプロイを管理」→ 鉛筆マーク →「バージョン」を 新バージョン にして「デプロイ」を押してください。URL は変わりません。

大会運用

大会前日

setupTrigger を実行します。2分ごとの自動生成が有効になります。

大会終了後

removeTrigger を実行します。自動生成が停止します。

開発テスト用リポジトリに切り替える場合は、スクリプトプロパティの GITHUB_REPO をテスト用リポジトリ名に変更してください。本番に戻すときは元の値（{{GITHUB_REPO}}）に戻してください。

GITHUB_TOKEN を変更しても、すでに記録されたハッシュはそのまま有効です。念のため再生成対象をリセットしたい場合は、関数の一覧から testClearAllHashes を実行してください。
