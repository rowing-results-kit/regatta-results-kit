Features

3つの特徴

大会担当者が「運用できる」ことを最優先に設計しています。

⚙️

設定1枚で一発生成

大会情報を入力したら、速報サイト・判定員帳票・運用手順書まで自動で出来上がります。 毎大会コピーして再設定するだけ。テンプレートを何もないところから作る必要はありません。

📂

当日は CSV を入れるだけ

計測システムの CSV を Google Drive の指定フォルダに入れると、 2分おきに自動取り込みが走り、数分以内（最短2〜3分）に観客のスマホへ反映されます。 git の操作は一切不要です。

🗂️

年度ハブで大会をまとめる

協会の1年間の大会を一覧ページに束ねられます。 結果は大会ごとに永久アーカイブとして残り、 翌年以降も参照できます。

Get started

採用手順 4ステップ

最短1日で本番環境が整います。当日担当者に手順書を渡せば、あとは任せられます。

1

GitHub でテンプレートから
自分のリポジトリを作成

⏱ 約5分 — 無料アカウントでOK

GitHub の「Use this template」ボタンを押して、自団体専用のリポジトリを作成します。 アカウントを持っていなければ、その作成も含めて5分でできます。

2

ウィザードに大会情報を答える

⏱ 約10分

大会名・日程・会場・距離などを入力するだけ。 Claude Code を使っていれば /regatta-setup と入力するだけで自動化されます。

3

生成されたサイトを
Cloudflare Pages に接続

⏱ 約10分 — 無料

Cloudflare Pages の管理画面でリポジトリを選択してつなぐだけ。 以後はファイルを更新すると自動でデプロイされます。

4

Google スプレッドシートと
GAS を手順書どおりに接続

⏱ 約30分 — 大会固有値入りの手順書が自動生成

scaffold 実行時に手順書が自動生成されます。 設定値が埋め込み済みの手順書どおりに操作すれば、 エンジニアでなくても設定できます。

Integration

「リンクを入れるだけ」連携

大会サイトと年度ハブは、それぞれのファイルに URL を書き加えるだけでつながります。

🗂️ 大会をハブに載せる

hub/association.json に1エントリ追記するだけ。

{ "id": "spring-2026", "name": "春季ローイング選手権", "year": 2026, "dates": ["2026-05-10"], "status": "upcoming", "url": "https://spring-2026.pages.dev" }

↩️ 大会サイトからハブへ戻る

tournament.config.json の hub_url にハブのURLを入れるだけ。

{ "hub_url": "https://your-hub.pages.dev" }
指定の場所に URL を貼るだけで、お互いのページがつながります。
大会サイトのヘッダーに「◀ 大会一覧へ」リンクが自動的に表示され、
ハブには大会カードとステータスバッジ（開催予定・速報中・結果確定）が表示されます。

FAQ

よくある質問

費用はかかりますか？ ▼

GitHub・Cloudflare Pages・Google スプレッドシート・Google Apps Script（GAS）、 すべて無料枠での運用が可能です。 月間アクセス数や利用状況によって有料プランへのアップグレードが推奨される場合もありますが、 通常の大会規模であれば無料枠で十分です。

プログラミングの知識は必要ですか？ ▼

初期構築（テンプレート複製・Cloudflare 接続・GAS 設定）は、 設定値が埋め込み済みの手順書どおりに、コピー&ペースト中心の操作だけでできます。 大会当日の運用は、計測 CSV を Google Drive の指定フォルダにドラッグ&ドロップするだけです。 コードを書く必要はありません。

データは誰のものになりますか？ ▼

すべての結果データは、みなさん自身の GitHub リポジトリと Google ドライブに残ります。 Google Drive・スプレッドシート・GAS はみなさん自身の Google アカウント上に作成して使います。 キット提供者のアカウントやストレージを共有する仕組みは一切なく、 インフラもデータも 100% みなさんの所有です。

大会が終わったらサイトはどうなりますか？ ▼

サイトはそのまま結果アーカイブとして残り続けます。 「来年も同じ大会を開催する」場合は設定ファイルをコピーして年度だけ更新すれば再利用できます。 また、年度ハブからリンクが貼られているため、過去大会の結果を関係者がいつでも参照できます。

サイトの色は変えられますか？ ▼

管理者ポータルの「デザイン」タブでプレビューしながら変更・反映できます。 カラーピッカーでブランド色（プライマリ・アクセント）を選ぶと、ポータル内のミニプレビューに即時反映されます。 「公開サイトに反映」ボタンをクリックするだけで設定が保存され、数分以内に速報サイトの色が変わります。 コードの編集や git の操作は一切不要です。

まず、テンプレートを手元に置いてみてください。

GitHub でフォークするだけ。費用はかかりません。
